# Deduplication

## 简介

一般的重删技术分两大类，在线重删和post-process deduplication, 概念的东西不赘述了。直接上结论，XlatorS 采用的在线重删，原因来自于它的架构，post-process 不是不能做，而是没必要，会给存储系统增加额外大量的开销，这不是一个翻译该做的事了

在线重删在二级存储（备份/冷数据）领域应用极广，会给一下方面带来好处：

1. 减少存储空间的使用
2. 提高针对重复数据的传输带宽（ThroughPut）
3. 可以多版本保留

但也带来很多挑战

1. 元数据的管理
2. 小数据切块的整合及在磁盘空间的管理
3. 指纹的缓存/查询
4. 分布式系统
5. 数据修改引来的引用问题
6. 数据加密等
